# 分割消融实验结果分析正文稿

基于 `v2` 验证集，我们对分割模型的关键模块进行了系统消融，并在 `3` 个随机种子（`42 / 52 / 62`）下统计均值结果。完整模型 `A0` 取得了 `48.64\%` 的 `Leak IoU`、`73.73\%` 的 `mIoU` 和 `65.44\%` 的 `F1`。总体来看，除 `noise guidance` 外，其余被移除模块都会带来稳定性能下降，说明当前模型的主要性能收益并非来自单一技巧，而是由多尺度建模、渐进式分割、相关性校正以及边界分支概率引导共同构成。

其中，多尺度结构的贡献最为显著。当多尺度模块退化为单尺度 `k=[20]` 后，`Leak IoU` 从 `48.64\%` 降至 `44.51\%`，平均下降 `4.13` 个百分点，`mIoU` 和 `F1` 也分别下降 `2.12` 和 `3.85` 个百分点，且 `3` 个种子下均呈现一致退化。这表明多尺度建模并非简单的冗余设计，而是当前模型识别稀疏 leak 区域的核心能力来源；去掉该模块后，precision 与 recall 同时下降，说明模型整体表征能力受损，而不是仅仅出现阈值偏移。

相比之下，渐进式分割结构、噪声-渗漏相关性校正以及边界分支中的 fine 概率输入都呈现出中等幅度但稳定的正向贡献。去掉渐进式结构后，`Leak IoU` 平均下降 `0.69` 个百分点；去掉相关性校正后，`Leak IoU` 下降 `0.83` 个百分点，并伴随 recall 的明显下降；边界分支不再使用 fine 概率输入时，`Leak IoU` 进一步下降 `0.95` 个百分点。由于这些变体在 `3` 个种子下均为一致退化，因此可以认为上述三个模块提供的是可重复、可验证的稳定收益，而非偶然的训练波动。

需要注意的是，`noise guidance` 的作用在当前实验设置下并未表现出稳定增益。去掉该模块的 `A1` 在均值上略优于完整模型，`Leak IoU` 提升约 `0.30` 个百分点，但其标准差也显著增大，且 `3` 个种子的 paired 变化符号为 `++-`，即并非一致改善。该结果说明，`noise guidance` 目前更可能是一个对训练超参数、采样策略和损失加权较为敏感的模块，而不是已经被充分验证的稳定贡献项。因此，在论文表述中，更合适的说法应是其效果“尚不稳定，有待进一步验证”，而不宜直接将其归纳为关键有效模块。

此外，需要强调的是，本节消融实验中的完整模型 `A0` 采用的是统一消融 protocol，即 `200 epochs`、`small_positive` 采样策略、类别加权分割损失以及基于 `val_global_leak_iou` 的模型选择准则；因此，它与主实验部分使用的 `300 epochs + shuffle + no class weights` baseline 不完全等价。两者结果接近，但在论文中应避免将两套 protocol 的数值直接混写到同一结论中，以保证实验对比的严谨性。
