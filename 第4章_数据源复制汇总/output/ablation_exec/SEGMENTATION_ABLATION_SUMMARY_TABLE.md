# 分割消融实验总结表

适用场景：Word 论文、PPT 汇报、答辩备注  
统计口径：3 个 seed（`42 / 52 / 62`）均值；`Δ` 为相对完整模型 `A0` 的 paired delta

## 一页讲清版

| 模块 / 变体 | 结论 | 关键证据 | 影响级别 | 汇报时建议说法 |
| --- | --- | --- | --- | --- |
| `A2` 去掉 multi-scale | 明显退化，且退化最稳定 | `Leak IoU -4.13`，`mIoU -2.12`，`F1 -3.85`；3 个 seed 全为负 | 极大 | 多尺度结构是当前分割模型的核心贡献之一，去掉后 precision 和 recall 同时下降，不是简单阈值波动。 |
| `A5` 边界分支不用 fine 概率 | 稳定退化 | `Leak IoU -0.95`，`F1 -0.86`；3 个 seed 全为负 | 中等偏大 | fine 概率输入能够为边界分支提供有效的局部判别信息，尤其帮助降低漏检。 |
| `A4` 去掉 noise-leak corr | 稳定退化 | `Leak IoU -0.83`，`F1 -0.75`；3 个 seed 全为负 | 中等 | 相关性校正主要帮助提升 leak 类召回，不是简单提升精度。 |
| `A3` 去掉 progressive | 稳定退化 | `Leak IoU -0.69`，`F1 -0.63`；3 个 seed 全为负 | 中等 | progressive 结构能稳定改善整体分割质量，尤其在 suppress 误报方面有帮助。 |
| `A1` 去掉 noise guidance | 均值略升，但不稳定 | `Leak IoU +0.30`，但标准差最大（`±0.87`），seed 符号为 `++-` | 不确定 | 当前证据不足以说明 noise guidance 是稳定有效模块，它更像是依赖训练设定的敏感项，需要继续验证。 |

## 备注

- 这套消融的 `A0` 是统一 protocol 下的完整模型，不等于你之前单独汇报的那版 `300 epochs + shuffle + no class weights` baseline。
- 当前最稳的结论是：`multi-scale` 必须保留，`progressive / corr / fine-prob boundary` 都是稳定正贡献。
- 当前最需要继续验证的模块是 `noise guidance`，因为它没有表现出跨 seed 的稳定收益。
